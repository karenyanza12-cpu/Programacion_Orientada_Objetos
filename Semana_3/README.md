# Tarea Semana 3 - Programación Orientada a Objetos

## Estudiante

**Nombre Completo:** Karen Yanza

---

## Descripción

Este proyecto contiene dos programas que resuelven el mismo problema: registrar y mostrar información básica de una mascota.

Los datos registrados son:
- Nombre
- Especie
- Edad

---

## Programa Tradicional

**Ubicación:** `programacion_tradicional/tradicional.py`

**Características:**
- Utiliza funciones.
- Utiliza variables.
- Solicita datos por teclado.
- No utiliza clases ni objetos.

---

## Programa Orientado a Objetos

**Ubicación:** `programacion_poo/` (Archivos: `mascota.py` y `main.py`)

**Características:**
- Utiliza una clase llamada Mascota.
- Implementa atributos.
- Implementa métodos.
- Crea objetos de la clase.
- Muestra la información de cada mascota.

---

## Reflexión

En esta práctica comprobé que la Programación Tradicional permite resolver problemas simples mediante funciones y variables, pero conforme el programa crece se vuelve más difícil organizar la información.

Por el contrario, la Programación Orientada a Objetos me permitió agrupar de forma lógica los datos y comportamientos dentro de clases y objetos. En este ejercicio quedó en evidencia cómo la clase Mascota facilita una gestión de la información mucho más ordenada, reutilizable y fácil de mantener a través de sus atributos y métodos.
